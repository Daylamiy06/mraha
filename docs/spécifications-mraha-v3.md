SPÉCIFICATION MRAHA WA TSO

1. DÉFINITIONS

1.1 Joueur actif

Le joueur actif est le joueur dont c'est actuellement le tour.

1.2 Rangée intérieure

Chaque joueur possède huit cases intérieures numérotées :

1 2 3 4 5 6 7 8

1.3 Rangée extérieure

Chaque joueur possède huit cases extérieures numérotées :

9 10 11 12 13 14 15 16

1.4 TSO

La phase TSO est la première phase de la partie.

Pendant cette phase :

chaque joueur possède encore des graines en réserve ;
chaque tour commence par le dépôt obligatoire d'une graine de réserve dans une case intérieure non vide.

1.5 NDRAZI

La phase NDRAZI commence lorsque les deux réserves sont vides.

Pendant cette phase :

les coups sont joués uniquement avec les graines présentes sur le plateau.

1.6 Capture

Une capture est possible lorsque :

la dernière graine d'un semis tombe dans une case intérieure contenant déjà au moins une graine du joueur ;
la case adverse située en face contient au moins une graine adverse.

Dans ce cas :

seules les graines adverses sont capturées ;
les graines du joueur restent dans leur case.

1.7 Direction active

La direction active vaut toujours :

1
ou
8

Elle détermine :

le point de départ utilisé après chaque capture.

1.8 Mkazo

Un Mkazo est un coup sans capture.

Le joueur sème ses propres graines jusqu'à ce que la dernière graine tombe sur une case vide.

1.9 Udza

L'Udza est une annulation volontaire d'un Mkazo.

1.10 Nyumba

Le Nyumba est la case intérieure numéro 5.

Le Nyumba possède des règles particulières décrites au chapitre correspondant.

Un semis dépose :

exactement une graine par case traversée.

La case de départ est vidée avant le semis.

Le semis commence toujours à la case suivante.

4. PHASE TSO

4.1 Début obligatoire du tour

Lorsque c'est son tour, le joueur :

retire exactement une graine de sa réserve ;
choisit une case intérieure non vide de son camp.

Une case vide ne peut jamais recevoir une graine de réserve.

4.2 Recherche obligatoire des captures

Avant de rechercher un Mkazo, le joueur doit rechercher toutes les captures possibles.

Une capture est possible lorsque :

la graine de réserve est déposée dans une case intérieure non vide ;
après ce dépôt, la case adverse opposée contient au moins une graine.

Si au moins une capture existe :

une capture est obligatoire ;
aucun Mkazo n'est autorisé.

4.3 Déclenchement de la capture

La graine de réserve est déposée dans la case choisie.

Cette graine constitue la dernière graine du mouvement initial.

La capture est alors immédiatement évaluée.

4.4 Direction active initiale en TSO

Après le dépôt de la graine de réserve :

Dépôt dans la case 1 ou 2
direction active = 1
Dépôt dans la case 7 ou 8
direction active = 8
Dépôt dans la case 3, 4, 5 ou 6

Le joueur choisit :

direction active = 1
ou
direction active = 8

Cette direction devient la direction active du tour.

4.5 Capture

Lorsqu'une capture est exécutée :

seules les graines adverses sont capturées ;
les graines du joueur restent dans leur case.

Les graines capturées sont ensuite semées.

Le semis repart toujours depuis :

1
ou
8

selon la direction active.

4.6 Évaluation après chaque semis

Après chaque semis :

Cas 1 : la dernière graine tombe sur une case vide

Le tour prend fin immédiatement.

Cas 2 : la dernière graine tombe sur une case contenant déjà au moins une graine

Une capture éventuelle doit être examinée.

4.7 Capture en chaîne

Si la dernière graine tombe dans une case intérieure et que :

la case adverse opposée contient au moins une graine ;

alors :

la capture est obligatoire ;
seules les graines adverses sont capturées ;
les graines capturées sont semées depuis la direction active.

4.8 Absence de capture

Si le tour a commencé par une capture et qu'aucune capture n'est possible :

le joueur prend toutes les graines présentes dans la case où la dernière graine vient de tomber ;
il poursuit immédiatement le semis depuis la case suivante.

Cette chaîne continue jusqu'à :

une capture ;
ou une case vide.

4.9 Mise à jour de la direction active

La direction active reste inchangée tant qu'une capture est exécutée sur une case intérieure non extrême.

Si une capture est exécutée sur :

1 ou 2

alors :

direction active = 1

Si une capture est exécutée sur :

7 ou 8

alors :

direction active = 8

4.10 Capture d'une seule graine

Aucune règle spéciale n'existe.

Une seule graine capturée est semée exactement comme plusieurs graines capturées.

La seule différence est qu'une unique graine capturée devient immédiatement la dernière graine du nouveau semis.

Après son dépôt :

une capture éventuelle doit être examinée ;
si une capture existe, elle est obligatoire ;
sinon le joueur reprend toutes les graines présentes dans la case où cette graine vient de tomber et poursuit immédiatement le semis.

Exemple

La direction active vaut :

8

Une capture permet de prendre exactement :

1 graine adverse

Cette graine est semée depuis :

8

Comme il s'agit de l'unique graine capturée :

elle est immédiatement la dernière graine du semis ;
une nouvelle capture éventuelle doit être examinée.

Si aucune capture n'est possible :

le joueur prend toutes les graines présentes dans la case 8 ;
le semis continue depuis la case 7.


5. TRANSITION TSO → NDRAZI

5.1 Réserves synchronisées

Au début de la partie :

chaque joueur possède 22 graines ;
chaque tour TSO consomme exactement une graine.

Par conséquent :

les réserves restent toujours synchronisées.

5.2 Dernier tour TSO

Lorsqu'un joueur joue sa dernière graine :

son tour se termine normalement ;
l'adversaire joue ensuite son propre dernier tour TSO.

5.3 Début officiel de NDRAZI

Après ce dernier tour :

les deux réserves sont vides ;
la phase TSO se termine ;
la phase NDRAZI commence immédiatement.

6. PHASE NDRAZI

6.1 Principe général

Pendant la phase NDRAZI :

aucune graine n'est ajoutée depuis une réserve ;
tous les coups sont joués avec les graines du plateau.

6.2 Définition d'un Ndrazi

Un Ndrazi est un coup qui provoque une capture.

Un Ndrazi peut commencer :

depuis une case intérieure ;
depuis une case extérieure.

Dans tous les cas :

la case de départ doit contenir au moins deux graines ;
le coup doit produire une capture.

6.3 Priorité absolue

Avant chaque tour :

le joueur doit rechercher tous les Ndrazi possibles.

Si au moins un Ndrazi existe :

un Ndrazi est obligatoire ;
aucun Mkazo n'est autorisé.

6.4 Choix du Ndrazi

Si plusieurs Ndrazi existent :

le joueur choisit librement lequel il souhaite jouer.

6.5 Choix du sens initial

Lorsqu'un Ndrazi est sélectionné :

si un seul sens produit une capture, ce sens est obligatoire ;
si les deux sens produisent une capture, le joueur choisit librement.

Un Ndrazi reste un seul coup même si les deux sens sont possibles.

6.6 Détermination de la direction active initiale

Le sens choisi détermine immédiatement la direction active.

Sens gauche → droite
direction active = 1
Sens droite → gauche
direction active = 8

6.7 Ndrazi démarrant depuis l'extérieur

Un Ndrazi peut commencer dans une case extérieure.

Le joueur choisit un sens légal.

Entrée par la case 1

La direction active devient :

1

Entrée par la case 8

La direction active devient :

8

Exemple :

12 → 11 → 10 → 9 → 8 → 7 → 6

La dernière graine tombe sur 6.

Si une capture est exécutée :

les graines capturées seront semées depuis 8 ;
parce que le semis est entré par la droite.

6.8 Capture

Lorsqu'une capture est exécutée :

seules les graines adverses sont capturées ;
les graines du joueur restent dans leur case.

6.9 Reprise après capture

Les graines capturées sont semées.

Le semis repart toujours depuis la direction active.

6.10 Évaluation après chaque semis

Après chaque semis :

Dernière graine sur case vide

Le tour prend fin.

Dernière graine sur case non vide

Une capture éventuelle doit être examinée.

6.11 Captures en chaîne

Si une nouvelle capture existe :

elle est obligatoire ;
elle est exécutée immédiatement.

6.12 Absence de capture

Si le tour a commencé par une capture et qu'aucune capture n'est possible :

le joueur prend toutes les graines présentes dans la case où la dernière graine vient de tomber ;
il poursuit immédiatement le semis depuis la case suivante.
6.13 Mise à jour de la direction active

La direction active reste inchangée tant qu'une capture est exécutée sur une case intérieure non extrême.

Si une capture est exécutée sur :

1 ou 2

alors :

direction active = 1

Si une capture est exécutée sur :

7 ou 8

alors :

direction active = 8

6.14 Capture d'une seule graine

Aucune exception n'existe.

Les mêmes règles s'appliquent toujours.

Une seule graine capturée est semée exactement comme plusieurs graines capturées.

La seule différence est qu'une unique graine capturée devient immédiatement la dernière graine du nouveau semis.

Après son dépôt :

une capture éventuelle doit être examinée ;
si une capture existe, elle est obligatoire ;
sinon le joueur reprend toutes les graines présentes dans la case où cette graine vient de tomber et poursuit immédiatement le semis.
Exemple

La direction active vaut :

1

Une capture permet de prendre exactement :

1 graine adverse

Cette graine est semée depuis :

1

Elle tombe donc dans la case :

1

Comme il s'agit de l'unique graine capturée :

elle est immédiatement la dernière graine du semis ;
une nouvelle capture éventuelle doit être examinée.

Si aucune capture n'est possible :

le joueur prend toutes les graines présentes dans la case 1 ;
le semis continue depuis la case 2.

7. NYUMBA

7.1 Définition

Le Nyumba est la case intérieure numéro 5.

Au début de la partie :

cette case possède le statut spécial Nyumba.

7.2 Conservation du statut

Le statut Nyumba reste actif tant que la case 5 n'a jamais servi de case de départ.

Recevoir des graines ne retire jamais le statut.

7.3 Perte du statut

Le statut Nyumba est perdu définitivement lorsque la case 5 sert de case de départ.

Après cela :

la case devient une case normale ;
le statut Nyumba ne peut jamais être récupéré.

7.4 Dépôt interdit pour un Mkazo TSO

Pendant la phase TSO :

il est interdit de déposer une graine de réserve dans le Nyumba pour effectuer un Mkazo.

7.5 Capture autorisée depuis le Nyumba en TSO

Cette interdiction disparaît si le dépôt produit immédiatement une capture.

Dans ce cas :

le dépôt est autorisé ;
la capture est exécutée normalement.

7.6 Nyumba comme Ndrazi

Pendant la phase NDRAZI :

le Nyumba participe normalement à la recherche des Ndrazi.

Si :

le Nyumba contient au moins deux graines ;
un Ndrazi légal peut être joué depuis cette case ;

alors :

ce Ndrazi est autorisé ;
le joueur peut le choisir comme n'importe quel autre Ndrazi.

7.7 Cas spécial du Nyumba isolé

Cette règle n'existe que si :

en phase TSO :

aucune capture n'est possible ;

ou

en phase NDRAZI :

aucun Ndrazi n'est possible ;

et

aucune autre case intérieure ne contient de graines.

Phase TSO

Le joueur :

retire une graine du Nyumba ;
retire une graine de sa réserve ;
sème les deux graines.

Vers la gauche :

4 puis 3

Vers la droite :

6 puis 7

Le statut Nyumba est conservé.

Phase NDRAZI

Le joueur :

prend toutes les graines du Nyumba ;
choisit un sens ;
effectue un Mkazo normal.

Le statut Nyumba est perdu.

7.8 Arrêt volontaire dans le Nyumba (TSO uniquement)

Cette règle n'existe qu'en phase TSO.

Si :

la dernière graine tombe dans le Nyumba ;
aucune capture n'est exécutée à cet instant ;

alors le joueur peut :

Arrêter le tour

le tour prend fin ;
le statut Nyumba est conservé.

Continuer le semis

le joueur reprend les graines du Nyumba ;
poursuit le semis normalement ;
perd définitivement le statut Nyumba.

7.9 Capture prioritaire

Si :

la dernière graine tombe dans le Nyumba ;
la case adverse opposée contient au moins une graine ;
le tour a commencé par une capture ;

alors :

la capture est obligatoire ;
l'arrêt volontaire n'est pas autorisé.


8. MKAZO

8.1 Définition

Un Mkazo est un coup joué lorsqu'aucune capture obligatoire n'est disponible.

Le Mkazo consiste à prendre les graines d'une case légale puis à les semer selon les règles normales de circulation du plateau.

Le Mkazo existe :

pendant la phase TSO ;
pendant la phase NDRAZI.
8.2 Priorité de recherche
Phase TSO

Le joueur doit d'abord rechercher une capture.

Si au moins une capture existe :

la capture est obligatoire ;
aucun Mkazo n'est autorisé.

Le Mkazo n'est autorisé que lorsqu'aucune capture n'est possible.

Phase NDRAZI

Le joueur doit d'abord rechercher un Ndrazi.

Si au moins un Ndrazi existe :

le Ndrazi est obligatoire ;
aucun Mkazo n'est autorisé.

Le Mkazo n'est autorisé que lorsqu'aucun Ndrazi n'existe.

8.3 Conditions de validité
Pendant la phase TSO

Le joueur doit déposer une graine de réserve dans une case intérieure non vide.

Après ce dépôt :

la case doit contenir au moins deux graines ;
le Mkazo peut commencer.

Une case intérieure vide ne peut jamais être utilisée.

Pendant la phase NDRAZI

La case choisie doit déjà contenir au moins deux graines.

Une case contenant une seule graine ne peut jamais servir de départ pour un Mkazo.

8.4 Priorité intérieur / extérieur

Pendant la phase NDRAZI :

les Mkazo intérieurs sont recherchés en premier ;
les Mkazo extérieurs sont recherchés uniquement si aucun Mkazo intérieur n'existe.

Si au moins un Mkazo intérieur existe :

tous les Mkazo extérieurs sont interdits.
8.5 Choix du sens

Le joueur choisit librement le sens initial du Mkazo.

Cette liberté existe :

depuis une case intérieure ;
depuis une case extérieure ;
depuis une case extrême ;
depuis une case non extrême.

Aucune règle de direction active ne s'applique aux Mkazo.

Une fois le sens choisi :

il reste inchangé pendant tout le tour ;
le joueur doit conserver ce sens jusqu'à la fin du Mkazo.
8.6 Début du semis

Le joueur prend toutes les graines de la case de départ.

La case de départ devient vide.

Le semis commence dans la case suivante du sens choisi.

La case de départ ne reçoit jamais immédiatement une graine.

Exemple TSO

La case 8 contient 1 graine.

Le joueur dépose une graine de réserve.

La case contient maintenant :

2 graines

Le joueur prend ces 2 graines.

Il choisit le sens extérieur.

Le semis commence :

9 puis 10
8.7 Reprise des graines

Après chaque semis :

Si la dernière graine tombe sur une case vide

Le tour prend immédiatement fin.

Si la dernière graine tombe sur une case non vide

Le joueur prend toutes les graines présentes dans cette case.

Il poursuit immédiatement le semis dans le même sens.

Cette opération peut se répéter autant de fois que nécessaire.

8.8 Interdiction absolue des captures

Un tour commencé par un Mkazo ne peut jamais effectuer une capture.

Cette règle est absolue.

Même si une situation capturable apparaît :

pendant le semis ;
après le semis ;
plusieurs fois pendant la chaîne ;

aucune capture n'est autorisée.

Le joueur applique uniquement la règle de reprise décrite à l'article 8.7.

8.9 Conservation du sens

Lorsqu'une reprise de graines se produit :

le sens initial reste inchangé ;
le joueur ne peut jamais changer de sens pendant le même Mkazo.

Toutes les reprises successives utilisent donc exactement le même sens de rotation.

8.10 Interdiction d'auto-élimination

Un Mkazo n'est légal que s'il ne provoque pas immédiatement la défaite du joueur.

Le moteur doit exclure tout Mkazo qui entraînerait directement :

une rangée intérieure vide ;
l'absence totale de Ndrazi ;
l'absence totale de Mkazo ;
l'absence totale de coup légal au tour suivant.

Un Mkazo qui conduit inévitablement à une position où le joueur sera en défaite dès son prochain tour est également interdit.

Cette situation est considérée comme une défaite d'avance et relève de la règle générale d'auto-élimination.

Exemple

La case intérieure 1 contient exactement 2 graines.

Toutes les autres cases intérieures sont vides.

Le joueur souhaite semer vers l'extérieur :

16 puis 15

La rangée intérieure devient entièrement vide.

Ce Mkazo est interdit.

8.11 Cas du Nyumba

Les règles particulières du Nyumba sont définies au chapitre 7.

Lorsqu'un Mkazo implique le Nyumba :

toutes les dispositions du chapitre 7 s'appliquent ;
aucune règle supplémentaire n'est créée dans ce chapitre.

8.12 Fin du tour

Le Mkazo prend normalement fin lorsque la dernière graine du semis tombe sur une case vide.

L'unique exception est le cas spécial du Nyumba défini à l'article 7.8, applicable uniquement pendant la phase TSO.

En dehors de cette exception, aucune autre condition ne peut mettre fin à un Mkazo.


9. UDZA

9.1 Définition

L'Udza est une annulation volontaire d'un Mkazo.

L'Udza est autorisé :

sur les Mkazo de la phase TSO ;
sur les Mkazo de la phase NDRAZI.

Un Ndrazi ou une capture ne peuvent jamais être annulés.

9.2 Disponibilité

Après l'exécution d'un Mkazo :

un bouton Valider apparaît ;
un bouton Udza apparaît.

Le joueur doit choisir l'une des deux actions.

9.3 Validation

Si le joueur choisit Valider :

le Mkazo devient définitif ;
le tour prend fin ;
le tour passe à l'adversaire.

9.4 Annulation

Si le joueur choisit Udza :

l'état complet du tour est restauré ;
le joueur revient à la position précédant le Mkazo.

La restauration comprend notamment :

toutes les graines ;
les réserves ;
le statut du Nyumba ;
le sens choisi ;
toutes les informations internes du moteur.

Le joueur peut alors choisir un autre Mkazo.

9.5 Limite

Chaque joueur dispose au maximum de :

2 Udza

par tour.

Après le premier Udza :

reste 1 Udza

Après le second Udza :

reste 0 Udza

9.6 Troisième essai

Lorsqu'un joueur joue un troisième Mkazo après avoir utilisé ses deux Udza :

aucun bouton Udza n'apparaît ;
aucun bouton Valider n'apparaît ;
le Mkazo est automatiquement validé.

Le tour passe immédiatement à l'adversaire.

9.7 Réinitialisation

Le compteur d'Udza est réinitialisé au début de chaque nouveau tour.

Chaque nouveau tour redonne :

2 Udza

au joueur actif.

10. DÉFAITE

10.1 Défaite pendant TSO

Pendant la phase TSO :

un joueur perd immédiatement lorsque sa rangée intérieure devient totalement vide.

Cette vérification est effectuée dès la fin du coup adverse ayant vidé la dernière case intérieure encore occupée.

L'animation du coup peut être affichée jusqu'à son terme pour des raisons visuelles, mais le résultat de la partie est déjà déterminé.

10.2 Défaite pendant NDRAZI

Pendant la phase NDRAZI :

un joueur perd immédiatement si :

aucun Ndrazi n'est possible ;
aucun Mkazo intérieur n'est possible ;
aucun Mkazo extérieur n'est possible.

Autrement dit :

aucun coup légal n'existe.

10.3 Défaite par auto-élimination

Un joueur ne peut jamais effectuer volontairement un coup qui provoque immédiatement sa propre défaite.

Ces coups sont interdits et doivent être exclus de la liste des coups légaux.

10.4 Défaite d'avance

Cette règle s'applique uniquement pendant la phase NDRAZI.

Lorsqu'aucun Ndrazi n'est disponible, le joueur peut être contraint de rechercher un Mkazo.

Un Mkazo est interdit s'il conduit immédiatement à une position dans laquelle le joueur ne disposera plus d'aucun coup légal lors de son prochain tour.

Cette situation est assimilée à une auto-élimination.

Le moteur doit donc exclure ces Mkazo de la liste des coups légaux.

Cette vérification ne nécessite aucune anticipation des coups adverses.

Elle repose uniquement sur la position résultant du Mkazo exécuté.

Si tous les Mkazo disponibles sont interdits par cette règle, le joueur est immédiatement déclaré perdant.

10.5 Abandon

Un joueur peut abandonner volontairement la partie.

L'abandon provoque immédiatement :

la fin de la partie ;
la victoire de l'adversaire.

10.6 Victoire

Lorsqu'un joueur est déclaré perdant :

l'adversaire est immédiatement déclaré vainqueur.

11. NOUVELLE PARTIE

11.1 Première partie

Lors de la toute première partie :

le premier joueur est désigné par tirage au sort.

11.2 Partie suivante

Après la fin d'une partie :

le vainqueur de la partie précédente devient automatiquement le premier joueur de la partie suivante.

11.3 Réinitialisation du plateau

Chaque nouvelle partie rétablit :

la position initiale ;
les réserves initiales ;
le statut initial du Nyumba ;
les compteurs d'Udza ;
tous les états internes du moteur.

11.4 Position initiale

La nouvelle partie recommence exactement avec la position officielle définie au chapitre 3.

Aucune information de la partie précédente n'est conservée.

11.5 Cycle de jeu

Le cycle complet d'une partie est :

Position initiale
→ Phase TSO
→ Phase NDRAZI
→ Défaite ou abandon
→ Victoire
→ Nouvelle partie


12 — API du moteur

12.1 Objectif

Le moteur constitue l'unique autorité sur les règles du jeu.

Il doit être totalement indépendant :

de l'interface graphique ;
des animations ;
du HTML ;
du CSS ;
des sons ;
du stockage.

Le moteur gère exclusivement :

l'état de la partie ;
les règles ;
les coups légaux ;
l'exécution des coups ;
l'historique ;
les annulations Udza ;
la détection des défaites ;
la détection du vainqueur.

L'interface ne doit jamais recalculer une règle du jeu.

12.2 Structure du plateau

Chaque joueur possède :

une rangée intérieure ;
une rangée extérieure.

Structure logique :

player = {
    inner: [8],
    outer: [8]
}

Chaque case contient un entier supérieur ou égal à zéro représentant le nombre de graines présentes.

12.2.1 Indexation interne

Les tableaux utilisent une indexation informatique commençant à zéro.

Correspondances :

inner[0] = case 1
inner[1] = case 2
inner[2] = case 3
inner[3] = case 4
inner[4] = case 5 (Nyumba)
inner[5] = case 6
inner[6] = case 7
inner[7] = case 8
outer[0] = case 9
outer[1] = case 10
outer[2] = case 11
outer[3] = case 12
outer[4] = case 13
outer[5] = case 14
outer[6] = case 15
outer[7] = case 16

Les numéros officiels des cases (1 à 16) restent utilisés dans :

la documentation ;
les journaux ;
l'interface utilisateur.

12.3 Structure d'un joueur

player = {
    inner: [8],
    outer: [8],

    reserve,
    nyumbaActive,
    udzaRemaining
}

12.5 Structure d'un coup

Chaque coup légal doit être entièrement décrit par une structure unique.

{

    direction
}

12.5.2 Mkazo TSO

{
    type: "TSO_MKAZO",

    targetPit,

    direction
}

12.5.3 Ndrazi

{
    type: "NDRAZI",

    startPit,

    direction
}

startPit désigne la case de départ du Ndrazi.

La case où débute la première capture est entièrement déterminée par les règles du jeu et par le sens choisi.

Elle ne fait donc pas partie des informations décrivant un coup.

12.5.4 Mkazo NDRAZI

{
    type: "NDRAZI_MKAZO",

    startPit,

    direction
}

12.5.5 Direction

Le moteur utilise uniquement :

LEFT
RIGHT

Correspondances :

RIGHT

correspond au sens :

1 → 2 → 3 → 4 → 5 → 6 → 7 → 8 → 9 → ...

LEFT

correspond au sens :

8 → 7 → 6 → 5 → 4 → 3 → 2 → 1 → 16 → ...

Les valeurs :

1
8

ne sont jamais stockées comme direction.

Elles ne sont qu'une conséquence des règles du jeu.

12.6 État sauvegardable

Pour permettre l'Udza, le moteur doit pouvoir restaurer exactement un état précédent.

les réserves ;

12.13 Principe fondamental

Le moteur est l'unique source de vérité.

L'interface :

ne calcule jamais les coups légaux ;
ne calcule jamais les captures ;
ne calcule jamais les Ndrazi ;
ne calcule jamais les Mkazo ;
ne calcule jamais les défaites ;
ne calcule jamais les transitions de phase.

Toutes les décisions doivent provenir du moteur.

L'interface ne fait qu'afficher l'état fourni par le moteur.

12.18 Exécution d'un coup

Le moteur doit fournir :

engine.play(move)

Cette fonction :

valide le coup ;
exécute le coup ;
applique les captures ;
applique les chaînes éventuelles ;
applique les pertes de statut Nyumba ;
applique les changements de phase ;
détecte les défaites ;
détermine le vainqueur ;
passe le tour au joueur suivant lorsque nécessaire ;
enregistre l'action dans l'historique.

La fonction retourne un objet décrivant entièrement le déroulement du coup.

12.18.3 Événements d'animation

Le champ :

réinitialise le compteur Udza du nouveau joueur.

Retour :

true

si la validation réussit.

Sinon :

false

12.20 Udza

canUndoMkazo()
engine.canUndoMkazo()

Retour :

true
false

Vérifie :

qu'il reste un Udza ;
que le dernier coup est un Mkazo.
undoMkazo()
engine.undoMkazo()

Effets :

restaure l'état précédent ;
consomme un Udza.

Retour :

true

si l'annulation réussit.

Sinon :

false

Lorsque le coup exécuté est un Mkazo, le moteur suspend temporairement la fin du tour.

Le résultat retourné par play() permet à l'interface d'afficher les animations puis de laisser le joueur choisir entre :

valider le Mkazo ;
utiliser un Udza.

Le tour n'est définitivement terminé qu'après cette décision.
